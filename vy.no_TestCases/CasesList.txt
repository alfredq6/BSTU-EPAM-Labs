Тесты проводятся на сайте https://www.vy.no/en/

Тест 1: Проверка поиска пустой строки по сайту

Шаги: открыть форму поиска, оставить поле поиска пустым, нажать на кнопку поиска;
Ожимадемый результат: поиск должен не выдать какие-либо результаты.


Тест 2: Проверка регистрации пользователя с пустыми полями

Шаги: открыть форму регистрации, оставить все поля пустыми, нажать на кнопку "save user and login";
Ожидаемый результат: новый пользователь не должен быть создан, выделяются поля, обязательные для заполнения и показываются соответствующие поля.


Тест 3: Проверка поиска маршрута без ввода пункта назначения/отправления

Шаги: на главной странице попытаться изменить поля "Where from" и "Where to", а затем оставить их пустыми;
Ожидаемый результат: кнопка поиска маршрутов должна быть неактивной, данные поля должны выделиться, а соответствующие ошибки должны быть описаны.


Тест 4: Проверка регистрации и входа корректного пользователя

Шаги: открыть форму регистрации, заполнить поля, нажать на кнопку "save user and login", в появившейся форме ввести email и пароль, нажать на кнопку "login and continue";
Ожидаемый результат: Открывается страница профиля вошедшего пользователя.


Тест 5: Проверка карты пользователя

Шаги: будучи незалогиненным, в форме ввода данных нужного маршрута нажать на кнопку "Add a Customer Card", ввести любой произвольный номер карты (не больше 8 символов), нажать на кнопку "View departures and fares";
Ожидаемый результат: Появление предупреждения, что карта с таким номером не распознана и тариф сменен на взрослый.


Тест 6: Проверка поиска расписания указанного маршрута

Шаги: в форме ввода данных нужного маршрута ввести данные, нажать на кнопку "View departures and fares";
Ожидаемый результат: вывод доступного расписания выбранного маршрута на выбранную дату и время.


Тест 7: Изменение забытого пароля зарегистрированного поля

Шаги: открыть форму входа в учётную запись, нажать на кнопку "Forgotten password", в появившемся окне ввести email, указанный при регистрации, в следующих окнах ввести полученный код и новый пароль;
Ожидаемый результат: вход пользователя с обновлённым паролем.


Тест 8: Проверка цены при оплате

Шаги: выбрать маршрут, в расписании выбрать один из них, перейти на страницу покупки билета, выбрать подходящий способ оплаты, нажать на кнопку "Continue to payment";
Ожидаемый результат: переход на страницу выбранного способа оплаты билета с соответствующей ему ценой.


Тест 9: Проверка максимального количества пассажиров на заказ

Шаги: при выборе маршрута попробовать указать больше 9 людей;
Ожидаемый результат: вывод предупреждения о превышении максимально возможного количества пассажиров и указание контакта для связи, чтобы организовать групповую поездку.


Тест 10: Проверка максимальной даты заказа

Шаги: при выборе маршрута попробовать указать дату оправления через 4 месяца;
Ожидаемый результат: Блокировка выбора следующего месяца и вывод предупреждения о максимальном количестве дней (90) с момента заказа.
